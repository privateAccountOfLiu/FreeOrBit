## 构建高级十六进制编辑器策划方案

### 1. 引言

010 Editor 是一款功能强大的十六进制编辑器，广泛应用于二进制分析、逆向工程、数据恢复等领域。其核心优势在于灵活的**二进制模板系统**、强大的**脚本支持**、高效的大文件处理以及丰富的数据分析功能。本方案旨在利用 Python + PySide6 技术栈，实现一款功能对标 010 Editor 的高级十六进制编辑器，提供专业的编辑、解析、分析能力。

### 2. 核心功能需求

#### 2.1 基础十六进制编辑
- **数据表示**：同时显示十六进制字节码、ASCII/Unicode 字符、十进制/其他进制数值。
- **基本操作**：插入、删除、覆盖、复制、粘贴（支持十六进制文本、原始字节、C数组格式等）。
- **光标与选择**：精确到字节/位的选择、块选择、列选择。
- **地址跳转**：支持绝对地址、相对偏移、书签跳转。
- **字节着色**：自定义字节范围高亮、结构体字段着色、搜索命中高亮。
- **编辑历史**：完整的撤销/重做栈，支持任意步数回退。

#### 2.2 高级编辑功能
- **填充/覆写**：使用固定值、模式（递增/递减）、伪随机数填充指定范围。
- **剪切/复制/粘贴**：支持跨文件、跨进程的二进制数据交换，格式包括原始二进制、十六进制文本、C/C++数组、Base64等。
- **字节操作**：按位运算（与、或、非、异或）、算术运算（加、减、乘、除）、位移、字节序转换（大端/小端）。
- **数据插入**：插入文件、插入模板生成的数据、插入重复模式。

#### 2.3 模板与结构解析
- **二进制模板系统**：
  - 定义结构化数据类型（类似 C 结构体），支持基本类型（int8/16/32/64, float, double, char, string）和复合类型（数组、结构体、联合体、位域）。
  - 支持条件分支、循环、函数等逻辑控制。
  - 模板脚本语言：可使用 Python 子集或专用语法，实现动态解析。
  - 实时解析与反馈：当光标移动或数据修改时，模板视图自动更新字段值。
- **结构视图**：树形展示解析出的结构体字段，支持字段编辑、跳转、导出。
- **变量监视**：显示当前作用域内的变量值，支持修改。
- **模板库**：提供常见文件格式（PE、ELF、BMP、PNG、ZIP 等）的预定义模板，用户可自定义、导入/导出。

#### 2.4 脚本支持
- **脚本引擎**：集成 Python 解释器（嵌入式 CPython），允许用户编写 Python 脚本操作数据、控制编辑器、自动化任务。
- **脚本功能**：
  - 访问和修改文件数据（读/写/搜索）。
  - 控制编辑器（光标移动、选区操作、视图切换）。
  - 调用内置函数（校验和、加密、解压缩）。
  - 与模板系统交互（动态生成模板、修改字段）。
  - 插件化扩展：脚本可作为插件注册到菜单、工具栏、快捷键。
- **脚本调试**：支持断点、单步执行、变量监视、输出日志。

#### 2.5 搜索与替换
- **搜索模式**：
  - 字节序列（十六进制字符串）。
  - 文本字符串（ASCII/Unicode，区分大小写/全词匹配）。
  - 数值范围（整型/浮点型，支持字节序）。
  - 正则表达式（针对文本区域）。
  - 掩码搜索（忽略某些位）。
- **搜索范围**：整个文件、选区、从光标到文件尾、所有打开的文件。
- **替换功能**：支持替换为固定字节、模板生成数据、脚本计算结果。
- **搜索结果列表**：显示所有匹配项的位置和上下文，支持双击跳转。
- **增量搜索**：随输入实时高亮匹配项。

#### 2.6 数据导入导出
- **导入**：
  - 从文件追加/插入/覆写。
  - 从十六进制文本（如 xxd 输出）、C 数组、Base64、其他编码导入。
  - 从模板定义生成数据（例如根据结构体定义生成二进制数据）。
- **导出**：
  - 选区导出为文件。
  - 导出为十六进制文本、C/C++/Python 数组、Base64、HTML/RTF 格式化报告。
  - 导出为结构化数据（JSON/XML/YAML，基于模板）。

#### 2.7 文件比较与同步
- **二进制比较**：以字节为单位显示两个文件的差异，支持差异高亮、同步滚动。
- **智能比较**：识别插入/删除导致的偏移变化，支持对齐比较。
- **差异合并**：将差异部分从一个文件复制到另一个文件。
- **三向比较**（可选）：比较三个文件版本，辅助合并。

#### 2.8 校验和与哈希
- **内置算法**：
  - 校验和：Checksum-8/16/32, CRC-8/16/32/64, Adler-32, Fletcher 等。
  - 哈希：MD5, SHA-1, SHA-2 系列, SHA-3, Blake2, Whirlpool 等。
- **自定义算法**：允许用户通过脚本定义新的校验算法。
- **实时计算**：选区或整个文件的校验值实时显示在状态栏或独立面板中。

#### 2.9 书签与注释
- **书签**：在指定地址设置书签，支持命名、颜色标记、快速导航。
- **注释**：为地址范围添加文本注释，支持导出为 JSON/XML 元数据。
- **书签面板**：列表展示所有书签和注释，支持搜索、过滤、跳转。

#### 2.10 性能优化（大文件处理）
- **内存映射**：使用 `mmap` 处理大文件（>2GB），避免一次性加载到内存。
- **分块加载**：对于超大文件，仅加载可见区域及邻近块，后台异步加载其他部分。
- **延迟解析**：模板解析仅在需要时进行，并缓存结果。
- **搜索优化**：使用 Boyer-Moore 等高效算法，支持后台搜索，实时取消。

#### 2.11 界面与用户体验
- **多标签页**：支持同时打开多个文件，独立工作区。
- **窗口布局**：可停靠/浮动面板（结构视图、脚本控制台、搜索结果、书签、校验和等）。
- **主题与配色**：内置亮色/暗色主题，支持自定义语法高亮颜色。
- **快捷键自定义**：所有操作可配置快捷键。
- **状态栏**：显示当前光标地址、选中范围大小、文件大小、模式（插入/覆盖）等。
- **右键菜单**：上下文相关操作（复制为十六进制、跳转到地址、添加书签等）。

#### 2.12 高级功能（扩展）
- **内存编辑**：附加到进程，直接编辑进程内存（需平台特定实现）。
- **磁盘编辑**：以管理员权限直接编辑物理磁盘/分区（需谨慎）。
- **撤销历史视图**：可视化撤销栈，可跳转到任意历史状态。
- **数据可视化**：将数据解释为图像（如原始像素）、音频波形等。
- **版本控制集成**（可选）：与 Git 集成，显示文件差异历史。

### 3. 技术架构设计

#### 3.1 整体架构
采用 **MVVM（Model-View-ViewModel）** 模式，使业务逻辑与界面解耦，便于测试和扩展。

- **Model**：二进制数据模型（`BinaryDataModel`），负责底层数据存储、读写、通知更新。
- **ViewModel**：提供面向视图的数据接口，封装选择、编辑命令、撤销/重做等状态。
- **View**：基于 PySide6 的 QWidget 派生类，包括十六进制视图、结构视图、控制台等。
- **Controller**：协调 Model 和 ViewModel，处理用户输入和应用程序逻辑。

#### 3.2 核心组件

| 组件                | 职责                                                                 | 技术选型                            |
| ------------------- | -------------------------------------------------------------------- | ----------------------------------- |
| 数据模型层          | 管理二进制数据，支持大文件映射，通知变更                             | Python `mmap`，自定义 `QAbstractTableModel` |
| 十六进制视图        | 自定义绘图，显示十六进制和 ASCII，处理键盘/鼠标交互                  | 继承 `QAbstractScrollArea`，自定义绘制 |
| 模板引擎            | 解析二进制模板，生成结构树，将结构体映射到数据地址                   | Python 嵌入（用户模板为 Python 脚本），或自定义 DSL |
| 脚本引擎            | 执行用户 Python 脚本，访问编辑器 API                                 | 嵌入式 CPython（`exec` + 受限命名空间） |
| 搜索模块            | 提供多种搜索算法，后台异步执行                                       | 使用 `QThread`，Boyer-Moore/KMP 等   |
| 比较模块            | 实现二进制差异比较，支持动态对齐                                     | 基于最长公共子序列（LCS）或 Myers 算法 |
| 校验和模块          | 内置多种算法，支持用户自定义                                         | Python `hashlib`, `zlib.crc32` 等    |
| 插件系统            | 加载脚本插件，注册菜单/工具栏                                        | 动态导入 Python 模块                 |
| 设置管理            | 保存/加载用户配置（快捷键、主题、最近文件等）                        | QSettings + JSON 文件                |
| 撤销/重做系统       | 记录数据变更命令，支持合并连续操作                                   | 命令模式（`QUndoStack`）             |

#### 3.3 关键技术点

- **大文件处理**：使用 `mmap` 将文件映射到内存，通过指针访问，避免复制。对于写入操作，通过 `mmap` 的写模式直接修改内存，并通知视图更新受影响的区域。对于超大文件，可采用分段加载策略，但 Python 的 `mmap` 在 64 位系统上能处理数 TB 文件，前提是虚拟地址空间足够。
- **自定义十六进制视图**：
  - 在 `paintEvent` 中绘制网格、字节、字符、高亮。
  - 计算每行显示的字节数（可配置），根据滚动位置计算可见区域。
  - 处理鼠标点击定位到字节，支持鼠标拖动选择范围。
  - 处理键盘输入（插入/覆盖模式），调用数据模型修改数据。
  - 使用 `QTextLayout` 辅助文本渲染，提高性能。
- **模板系统设计**：
  - 设计一个 Python 类 `BinaryTemplate`，用户编写的模板实际上是一个 Python 模块，其中定义了继承自 `TemplateField` 的类，每个类对应一个结构体。
  - 模板解析器：动态导入用户模板文件，实例化结构体类，通过反射获取字段信息，根据字段类型读取数据地址的值，生成树形结构。
  - 支持字段的读写：当用户在结构视图中修改某个字段值时，自动计算新值并写入数据模型。
  - 性能优化：只解析当前可见的结构区域，并缓存解析结果；当数据变化时，仅重新解析受影响的字段。
- **脚本引擎集成**：
  - 创建独立的 Python 解释器上下文，提供安全的 `globals` 和 `locals`。
  - 提供编辑器 API 对象（如 `editor` 全局变量），封装数据访问、光标操作、UI 控制等功能。
  - 使用 `PySide6.QtCore.QObject` 的 `pyqtSignal` 实现脚本与主线程的通信，确保 UI 更新在 GUI 线程。
  - 脚本执行在 `QThread` 中，防止阻塞界面，支持中止。
- **撤销/重做**：
  - 使用 `QUndoStack` 管理命令，每个编辑操作（修改、插入、删除、填充等）对应一个 `QUndoCommand` 子类。
  - 命令对象记录修改前后的数据范围，在 `redo`/`undo` 中调用数据模型更新。
  - 支持将连续的插入/删除合并为单个命令，减少栈条目。
- **搜索模块**：
  - 在 `QThread` 中执行搜索算法，通过信号返回结果。
  - 支持取消正在进行的搜索。
  - 对于字节序列搜索，可先转换为字节串，使用内置 `find` 或 Boyer-Moore 实现。
  - 对于正则表达式，仅应用于文本区域（ASCII/Unicode），二进制数据转字符串可能产生不可打印字符，需限制搜索范围。

### 4. 详细实现方案

#### 4.1 数据模型设计

定义 `BinaryDataModel` 类，继承 `QAbstractTableModel`（用于在表格视图中展示，但实际十六进制视图自定义绘制，此处仅为便于与标准视图集成）。主要方法：
- `__init__(self, file_path=None)`
- `load_file(self, file_path)`：使用 `mmap` 映射文件，若文件不存在则创建内存缓冲区。
- `data(self, index, role)`：返回指定行、列的数据（按字节）。
- `setData(self, index, value, role)`：修改单个字节，返回成功与否。
- `setDataRange(self, offset, data)`：批量写入数据，发射 `dataChanged` 信号。
- `insert_bytes(self, offset, count)`：在指定位置插入字节，对于 mmap 文件，需要重新映射或使用临时文件，但最好避免插入操作导致大文件整体移动。实际实现中，对于支持插入的编辑器，通常将文件加载到内存中处理（对超大文件不友好），但可设计为将插入点之后的数据复制到新位置，用 mmap 也可以实现（在文件末尾扩展，然后移动数据）。为简化，初期可以限制插入操作为内存编辑模式，当文件大于阈值时提示。
- `remove_bytes(self, offset, count)`：删除字节。
- `rowCount(self)`：返回总字节数。
- `columnCount(self)`：固定为 1（每单元格一个字节）或 16（每行16个字节）？模型应设计为按行存储，但自定义视图会直接根据偏移量请求数据，所以模型也可以提供按偏移量访问的方法。

更简洁的方式：模型只提供基础的字节数组访问接口，不继承 `QAbstractTableModel`。自定义视图直接持有模型引用，通过模型方法读写数据。但为了利用 Qt 的信号/槽机制，模型需继承 `QObject`。

#### 4.2 十六进制视图定制

自定义 `HexEditor` 类，继承 `QAbstractScrollArea`。主要成员：
- `model`：二进制数据模型引用。
- `bytes_per_line`：每行显示的字节数（可配置，如 16）。
- `row_height`：行高（像素）。
- `char_width`：字符宽度（等宽字体）。
- `cursor_pos`：当前光标位置（字节偏移）。
- `selection_start/end`：选区范围。
- `overwrite_mode`：插入/覆盖模式标记。
- 字体、颜色方案等。

重写方法：
- `paintEvent`：遍历可见行，为每个字节计算位置，绘制十六进制表示和 ASCII 表示。根据光标和选区绘制高亮背景。使用 `QFontMetrics` 计算文本位置。
- `mousePressEvent`/`mouseMoveEvent`：根据鼠标坐标计算字节索引，设置光标或选区。
- `keyPressEvent`：处理方向键、Home/End、PageUp/Down、Delete、Backspace、字符输入（在覆盖/插入模式下修改数据）。
- `resizeEvent`：根据宽度计算每行可容纳的字节数，更新 `bytes_per_line`。
- `wheelEvent`：滚动视图。

视图需要响应数据变化：连接到模型的 `dataChanged` 信号，重绘受影响区域。

#### 4.3 模板系统实现

##### 4.3.1 模板定义语言

用户模板以 Python 模块形式编写，例如：

```python
from binary_template import Struct, UInt32, Char, Array, String

class FileHeader(Struct):
    magic = Char[4]
    version = UInt32
    flags = UInt32

class Record(Struct):
    id = UInt32
    name = String(256)  # 固定长度字符串
    data = Array(UInt32, 10)

class MyTemplate(Struct):
    header = FileHeader
    record_count = UInt32
    records = Array(Record, lambda self: self.record_count)  # 动态数组
```

核心基类 `Struct` 提供：
- 字段定义（类属性）。
- 从数据偏移量读取字段的方法。
- 写入字段的方法。
- 生成树形结构的方法。

模板引擎动态加载用户模块，遍历所有 `Struct` 子类，实例化并绑定到数据模型，构建结构树。

##### 4.3.2 模板解析器

- `TemplateParser` 接收数据模型和根偏移量，构建字段树。
- 每个字段对应一个节点，节点包含：字段名、类型、值、起始偏移、大小。
- 树形结构通过 `QTreeView` 显示，支持编辑（双击值单元格，调用字段写入方法更新数据）。
- 当数据模型变化时，仅重新解析受影响的字段（通过监听数据变化信号，判断变化区域是否在某个字段范围内）。

##### 4.3.3 模板视图同步

- 使用 `QTreeView` 显示字段树。
- 自定义 `FieldItemDelegate` 提供值编辑。
- 用户点击字段节点时，在十六进制视图中定位到对应偏移并高亮。
- 实时更新：当数据被修改时，通过信号通知模板视图刷新字段值。

#### 4.4 脚本引擎

提供 `ScriptEngine` 类，管理 Python 解释器上下文，暴露 API 对象。

API 对象设计示例：

```python
class EditorAPI:
    def __init__(self, editor_widget):
        self.editor = editor_widget

    def get_data(self, offset, size):
        """读取原始字节"""
        return self.editor.model.read(offset, size)

    def set_data(self, offset, data):
        """写入字节"""
        self.editor.model.write(offset, data)

    def get_cursor(self):
        return self.editor.hex_view.cursor_pos

    def set_cursor(self, offset):
        self.editor.hex_view.set_cursor(offset)

    def show_message(self, text):
        QMessageBox.information(self.editor, "Script", text)

    # 更多 API...
```

脚本执行：

```python
def run_script(self, script_code):
    api = EditorAPI(self)
    globals = {"editor": api, "__builtins__": __builtins__}
    try:
        exec(script_code, globals)
    except Exception as e:
        self.show_error(str(e))
```

脚本可注册为菜单项：通过扫描脚本文件夹，为每个脚本创建一个 `QAction`，点击时执行。

#### 4.5 搜索与替换

- `SearchDialog` 对话框，提供各种搜索选项。
- `SearchWorker` 继承 `QThread`，执行搜索算法，通过信号返回匹配列表。
- 结果在 `SearchResultsDock` 中显示为表格，双击跳转。
- 替换操作：在用户确认后，对每个匹配项调用数据模型的 `setDataRange`。

#### 4.6 文件比较

- 实现 `BinaryCompare` 类，输入两个数据模型，计算差异列表。
- 差异计算：基于 LCS（最长公共子序列）或 Myers 算法，得到编辑脚本（插入、删除、替换）。
- 将差异转换为高亮区域，在比较视图中显示。
- 比较视图：并排显示两个十六进制视图，同步滚动，差异高亮。
- 合并功能：选择差异项，将一侧数据复制到另一侧。

#### 4.7 校验和与哈希

- 内置算法通过 Python 标准库实现。
- 提供 `ChecksumDialog`，用户选择算法和范围，实时显示结果。
- 用户自定义算法：通过脚本定义，脚本引擎可访问数据范围，返回校验值。

#### 4.8 撤销/重做

- 创建 `EditCommand` 基类，继承 `QUndoCommand`。
- 具体命令：`ModifyCommand`（记录旧值和新值范围）、`InsertCommand`、`DeleteCommand`、`FillCommand` 等。
- 数据模型在 `setDataRange` 等方法中创建命令并入栈。

#### 4.9 性能优化

- 使用 `mmap` 映射文件，避免将整个文件读入内存。
- 视图绘制时只绘制可见区域，使用双缓冲。
- 模板解析缓存：记录字段偏移，只重新计算受数据变化影响的字段。
- 搜索使用 `QThread`，避免界面卡顿。
- 对于超大文件的插入/删除，设计为临时文件辅助或提示用户转为内存编辑。

### 6. 挑战与风险

- **大文件性能**：Python 的 `mmap` 对超大文件支持良好，但插入/删除操作会导致文件重写，性能较差。需提供“内存编辑模式”提示，或设计更高效的数据结构（如分块链表）。
- **模板系统复杂性**：设计一个既灵活又易于使用的模板语言需要深思熟虑。可能先实现简单的类 C 结构体解析，再逐步增强。
- **脚本安全性**：用户脚本可能执行危险操作（如删除文件、访问系统资源）。应提供受限的执行环境（`__builtins__` 限制），并提示用户。
- **界面响应性**：后台搜索、模板解析需异步执行，避免阻塞 UI。合理使用 `QThread` 和信号。
- **跨平台兼容性**：Python + PySide6 支持 Windows、Linux、macOS，但进程内存编辑等功能需平台特定实现，可能需要条件编译。
- **撤销/重做与 mmap 的协调**：撤销操作需要保存修改前的数据，若直接修改 mmap，撤销时需恢复。可通过命令对象保存变更的原始数据（对于大范围修改可能内存开销大），或采用增量快照。

### 7. 总结

本策划方案详细阐述了使用 Python/PySide6 构建一款功能对标 010 Editor 的高级十六进制编辑器的功能需求、技术架构、实现细节和开发计划。通过合理利用 Python 生态（`mmap`、`hashlib`、`zlib`）和 PySide6 的 GUI 能力，可以实现一个高效、可扩展的二进制编辑工具。关键挑战在于大文件处理、模板系统设计和脚本集成，但通过分阶段迭代开发，可以逐步完善。最终产品将满足逆向工程、数据分析、嵌入式开发等领域的高级需求。

#### 8.3.1 进程模块地址显示（Cheat Engine 风格）

| 要点 | 说明 |
|------|------|
| **数据来源** | Windows：`EnumProcessModules` 枚举句柄 → 对每个模块 `GetModuleInformation` 取 `lpBaseOfDll` 与 `SizeOfImage` → `GetModuleBaseNameW` 取短文件名（如 `game.exe`、`ntdll.dll`）。 |
| **命中规则** | 对当前字节 VA `v`，若存在模块满足 `base ≤ v < base + SizeOfImage`，则 **RVA = v − base**；多区间异常重叠时取 **base 最大** 的命中模块（与常见调试器启发式一致）。 |
| **界面** | **Hex 左侧列**：命中 → `短文件名+0xRVA`（过长文件名截断）；未命中任何模块 → **页内偏移**（相对当前载入页，非负）。**状态栏**主行同步：`模块偏移` 或 `页内偏移`，并保留 **VA**、**页基址**、**主模块基址** 等附加行。 |
| **刷新** | 打开进程、**F5 刷新**、**跳转到 VA 并切换分页** 时重新枚举模块列表（DLL 可能加载/卸载）。 |
| **限制** | 仅 PSAPI 提供的已加载模块；**堆、映射内存、无 PE 头区域** 无法命中模块，只能显示页内偏移与绝对 VA；权限不足时列表为空并降级为仅页内偏移。 |

### 8.5 ORF（只读分析）

| 要点 | 说明 |
|------|------|
| **概念** | 在字节流上以 **滑窗步长 = 类型宽度**（小端 `struct` 解析）将窗口解释为 `i8/u8/i16…/f64` 等标量，按 **数值区间包含** 与 **排除区间** 筛选。 |
| **阅读框相位** | **文件绝对偏移 mod 类型宽度** 得到 `r∈[0,w)`（类比分子生物学阅读框）；匹配按 **r** 分组展示于左侧树。 |
| **界面** | 独立窗口（**工具 → ORF…**）：类型、最小/最大值、排除区间、可选「仅当前选区」；**左侧树形表**：按相位分组，列偏移 / 原始字节(hex) / 解析值；**右侧** **QPainter** **频数–偏移** 直方图（按绝对地址分箱，不强制依赖 QtCharts）。 |
| **性能** | 扫描在 **QThread** 中执行，主线程更新进度与分批结果；单区扫描上限与进程缓冲一致（如 **64MB**），表格展示可截断并显示总匹配数。 |
