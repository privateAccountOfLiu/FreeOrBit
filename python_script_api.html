FreeOrBit 脚本 API 参考

本文档描述「脚本」面板中用户 Python 代码可见的全局环境与 editor 对象。实现以源码 src/freeorbit/script/editor_api.py 为准；若与策划文档或其他材料不一致，以本页与源码为准。

概述

脚本由程序使用 exec(code, globals, globals) 执行，全局字典由 make_script_globals 构造。用户可用的顶层名称包括：

  • editor：当前活动文档的 EditorAPI 实例。
  • print：重定向到内部日志（与 editor.message 类似），输出在面板「输出」区域通过 log_text() 汇总显示。
  • __builtins__：受限的内置名集合（见下文），并非完整 Python 内置环境。

editor（EditorAPI）

类型：EditorAPI，绑定到当前标签页对应的 DocumentEditor。

方法说明
read(offset: int, size: int) -> bytes 从当前二进制模型读取从 offset 开始的 size 个字节。
write(offset: int, data: bytes) -> None 将 data 写入从 offset 起的区域。会先调用模型的 ensure_mutable_copy()，再 replace_range。
cursor() -> int 当前光标字节偏移（十六进制视图）。
set_cursor(offset: int) -> None 将光标设到 offset，半字节偏移为 0。
message(text: str) -> None 向脚本日志追加一行文本（字符串形式）。
log_text() -> str 返回当前脚本运行累积的日志，多行以换行符连接。

全局 print

脚本中的 print 被替换为自定义函数，将内容写入与 editor.message 相同的内部行缓冲：

  • 无关键字参数时：各参数以空格连接成一行追加。
  • 存在关键字参数时：行为为在参数字符串后附加 repr(kwargs)（用于提示不支持标准 print 的关键字参数）。

示例

# 读取开头 16 字节并在输出中显示十六进制表示
data = editor.read(0, 16)
editor.message(hex(data))

与程序内脚本面板占位提示一致。也可使用 print(hex(data))，输出同样进入「输出」区域。
