FreeOrBit 结构模板（Python）指南

本文档说明如何用 Python 为「结构」停靠面板编写自定义二进制模板。实现以 src/freeorbit/template/fields.py、structure_dock.py 为准；与「脚本」面板（python_script_api.html）是两套独立机制。

与脚本面板的区别

  • 脚本面板：通过 exec 运行代码，使用全局 editor（EditorAPI）读写与日志，适合一次性操作或批处理。
  • 结构模板：单独的 .py 文件，由程序动态加载，只需实现 build_field_tree(model)，返回一棵 FieldNode 树，用于在结构树中展示字段、偏移与可编辑值。

入口约定

模板模块必须提供以下函数（名称固定，与常量 TEMPLATE_ENTRY_FN 一致）：

def build_field_tree(model: BinaryDataModel) -> list[FieldNode] | None:
    ...
  • model：当前文档的 BinaryDataModel，可用 len(model)、model.read(offset, size) 等读取字节。
  • 返回值：顶层字段的列表；若返回 None 或未定义该函数，程序会回退到内置的 DWORD 预览树。
  • 若函数抛出异常、返回非列表、列表项不是 FieldNode、或 dtype 非法，程序会仍显示回退树，并在界面用红色文件名提示错误（详情见悬停提示）。

FieldNode

定义见 freeorbit.template.fields.FieldNode（数据类）：

字段说明
name 字段显示名。
offset / size 在文件中的字节偏移与长度；标量写回时长度必须与 dtype 一致。
value_repr 「值」列展示的字符串（可读格式）。
children 可选，嵌套的 FieldNode 列表，用于分组显示。
dtype 可选。非空且为允许的标量类型名时，该节点「值」列可编辑，并通过 encode_field_value 写回二进制。

允许的 dtype（写回）

与 VALID_DTYPES 一致，例如：u8、u16le/u16be、u32le/u32be、u64le/u64be、i32le/i32be、i64le/i64be、f32le/f32be、f64le/f64be。父节点若仅作分组，可将 dtype 设为 None。

freeorbit.template.builders

提供与 010 风格类似的便捷函数，从 model 读取并生成带正确 dtype 与 value_repr 的 FieldNode，例如 u8、u16_le、u32_le、i32_le、f32_le 等。可直接组合进 build_field_tree 的返回值。

加载与内置模板

  • 在「结构」面板使用加载选择任意 .py 文件；路径会记入设置，下次启动可恢复。
  • 内置菜单项来自包内目录 freeorbit/resources/templates/（随 wheel/源码分发），实现见 builtin_templates.py。
  • 在设置 → 外观与行为 → 结构模板中可配置：打开文件时按扩展名 / 文件头 Magic自动选用模板（多行规则，实现见 freeorbit/template/auto_template.py）；亦可启用「打开文件时自动加载上次模板」。
  • 仓库根目录下的 templates/example_build_field_tree.py 为完整示例，可复制后修改。

写回与校验

用户在「值」列修改后，程序调用 encode_field_value(dtype, text) 得到字节，再 replace_range。整数可输入十进制或 0x 前缀十六进制；浮点按 float 解析。若编码失败或长度与 size 不符，会提示并刷新树。

示例（节选）

from freeorbit.model.binary_data_model import BinaryDataModel
from freeorbit.template.builders import u16_le, u32_le
from freeorbit.template.fields import FieldNode

def build_field_tree(model: BinaryDataModel) -> list[FieldNode]:
    if len(model) < 8:
        return []
    return [
        u32_le("magic", 0, model),
        u16_le("version", 4, model),
    ]

完整示例见仓库 templates/example_build_field_tree.py。
